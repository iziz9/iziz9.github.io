how to make hexo blog

Posted on 2022-09-27 In How to

<<<hexo blog 만들기>>>
(hexo.io 의 documents - getting start 참고)

1.들어가기 전 환경설정하기
dev에서
npm -v 로 node.js 버전확인
$ npm install -g hexo-cli
(npm 인스톨 안될 경우 sudo명령 필요 or git bash를 관리자 권한으로 열어보기)

  1. 디렉토리 생성
    hexo init blog(와 같은 임의 디렉토리이름 지정)

ls 로 디렉토리 생성 확인
cd blog
ls로 디렉토리 내 파일 확인

  1. 포스팅
    hexo new post “My first github blog 등 임의 제목 입력”
    $ vi source/_posts/My-first-github-blog.md
    (( hexo.io 에서 front-matter참고 ))

categories:
“- aa”
“- bb”
등으로 카테고리 추가 가능

취소선 아래 포스트 내용작성
취소 후 다시 하고싶으면 ctrl+c 하고 다시쓰면 됨

hexo generate 로 브라우저 테스트
hexo server 로 로컬호스트에서 확인
ctrl+c로 서버종료

  1. 깃헙 저장소와 헥소 프로젝트 연결하기
    (( hexo.io 에서 one-command deployment참고 ))
    깃헙에서 레포지토리 새로만들기 / 레포지토리 이름을
    github의 유저네임 똑같이 작성.github.io로 설정

$ npm install hexo-deployer-git –save 로 설치
ls로 파일확인
vi _config.yml 로 들어가 작업

“# Site
title: Front-end Blog
subtitle: ‘hexo blog’
description: ‘블 로 그’
keywords:
author: iziz
language: en
timezone: ‘’

“# URL
“## Set your site url here. For example, if you use GitHub Page, set url as ‘https://username.github.io/project'
url: https://유저명.github.io
.
.
.
deployment 하단 내용도 바꾸기
deploy:
type: git
repo: https://github.com/유저명/유저명.github.io.git
branch: main

  • 주의
    주소는 https로 바꿔줘야 함
    github.io 뒤에 트레일링 슬래시 붙이지 않기

hexo clean 로 제너레이터 지우고
hexo deploy 다시

새로고침은 F5 말고 ctrl+shift+r로…
(캐시 초기화하고 새로고침됨)

  1. 테마바꾸기
    https://hexo.io/themes/
    접속해서 next 테마 선택 (또는 원하는 테마 선택)
    npm install hexo-theme-next

vi _config.yml 들어가서
theme next 로 바꿔줌

$ hexo clean && hexo generate
클린 -> 제너레이트 두가지 실행하라는 명령어

hexo server로 적용확인
ctrl+c로 서버 꺼줌

hexo clean && hexo deploy
깃헙 ctrl+shift+r 로 새로고침

완성!
